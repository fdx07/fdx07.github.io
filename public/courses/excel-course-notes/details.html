Tables

Tables are a very practical way of presenting data. They make the data easier to understand and better looking. Also, they have many features that integrate into formulas as special ranges.

  • To create a table: select a range -> Press Ctrl+L -> There you have it!
  • To convert a table into a normal range: select a cell within the table-> Go to Table Design tab -> From Tools-> Select “Convert to Range”

If Statements

  • If statements are used to adapt a single formula into multiple possible variants.
  • Formula: <=if(logical_test, [value_if_true], [value_if_false])>
  • Nested If statements usage: <=IF(I3="A", 5, IF(I3="B", 7.5, IF(I3="C", 8, 4.5)))>

  • Hiding cells in excel is done by hiding columns or rows. Select the rows / columns from the indicator bar and press right click then hide them.
  • To unhide, select the rows (under:on) / columns (left:right) surrounding the hidden values and press right click then unhide them.

  • To view the dates in a type you want (if this type doesn't exist already) choose the closest type to your will then go to custom section and modify the type you chose.

And Function

  • <=and(a, b, c)> this function generates “True” if all the conditions are true, and generates "False" if one or more of the conditions is false.
  • Normal usage: =and(G4 = “Full Time”, I4>7). If all the conditions are true generates: True, if they are not it generates: False.
  • Usage with 'if': =IF(AND(C2="Full Time", G2>=5),*(1+$K$4),IF(AND(C2="Part Time", G2>7), F2*(1+$K$5),F2))

Or Function

  • <=or(a, b, c)> this function generates “True” if one of the conditions is true, and generates "False" if all the conditions are false.
  • Normal usage: =or(G4 = “Full Time”, I4>7). If one of the conditions is true generates: True, if there isn't true condition it generates: False.
  • Usage with 'if': =IF(OR(B2 = "Mother", C2>5,F2="Yes"), "Deserves", "Doesn't Deserve")

Conditional Functions

Countif:

  • <=countif (range, criterion)> counts the cells that are in the range and fit the criterion you determined.
  • If you want to define a criterion like >3 that contains non-numeric characters, you must define it in quotation marks. There shouldn't be spaces between non-numeric characters and other characters.
  • Example:
    =COUNTIF(E5:E289, Q27)
    =COUNTIF(C:C, "Dead")

Countifs:

  • <=countifs([range1, criterion1], [range2, criterion2] ...)> counts the cells that are in the range and fit all the criteria you indicated.
  • In countifs() all ranges must contain the same number of cells.
  • Example:
    =COUNTIFS(C4:C288, "Alive", D4:D288, "Unemployed")
    =COUNTIFS(N:N, ">=1-6-2017", N:N, "<1-10-2017")

Sumif & Sumifs:

  • <=sumif(range, criterion, sum range)>
  • <=sumifs(sum range, criterion 1 range, criterion 1, criterion 2 range, criterion 2 ...)>
  • Utilizing sumif & sumifs, we can use sum function depending on the criterion / criteria.
  • Example:
    =SUMIF(D4:D288, P6, I4:I288)
    =SUMIFS(F4:F288, C4:C288, "Dead", J4:J288, "Yes")

Pivot Tables vs Conditional Functions

When you need to perform multiple and distinct analyses on a large dataset, pivot tables are an efficient and quick way to do so. However, if you are going to perform a limited number of analyses, it's better to use conditional functions.

Data Separation / Text to Columns

To separate a dataset of texts like names and surnames into columns we can use this feature. From Data> Data Tools> Text to Columns. This feature consists of 3 steps:

  • Data type: Here we determine if we want to separate the text using a sign it includes (delimited) or an alignment (fixed with) that appears between our data.
  • Separation place / sign: Here we choose the delimiter or the index that we want separation operation to happen.
  • Destination: Determine the destination.

Ampersand "&" and Concatenate Function

  • To merge two cells, we can use ampersand “&” symbol. Eg: = B2 & “ ” & C2
  • We can also use concatenate function: =concatenate (B2, “ ”, C2)
  • We can benefit from ampersand symbol using it in formulas too: =sum(A1:A5)&“ Students of " &sum(C1:C5)& “ groups.”

Goal Seek

This analysis method helps us to make some changes in our numbers so we can reach an aimed result. Eg: We have sold 75 laptops, 850$ per each, total 63750$. But we want to know how much per each we should purchase our clients to reach 70000$ in total. Here goals seek's role comes in. Data> What – If Analysis> Goal seek...

Vertical Lookup / VLOOKUP Function

  • <=vlookup (lookup value, table array, column index, range lookup)>
  • Lookup value: the value we are searching.
  • Table array: the range of our search. (The value's column must be the first column of the range.)
  • Column index: Column index number of the value we are looking for.
  • Range lookup: Searching method. True => Approximate match / False => Exact Match
  • This function brings us values that match the criteria from a specific search range.
  • Example:
    =VLOOKUP(A2, $L$2:$N$19, 2, FALSE)
    =VLOOKUP(G6, $A$2:$D$147, 2, FALSE)

IFERROR Function

  • <=iferror(value, value if error)>
  • The purpose of this function is being able to handle any error caused by any formula.
  • Example: =IFERROR(VLOOKUP(A2, $L$2:$N$19, 2, FALSE), "")

Macro

Macro is one of the most useful tools in excel. If we are applying a multiple step process repeatedly in an excel workbook, we must be familiar with macro.

  • Firstly, we go to View> Macros> Record Macro or we click to record sign at the left bottom corner then we give a name and a shortcut (optional) to our macro. Afterwards we choose where we want to use this macro. We have 3 options:
  • This workbook (Only): Choosing this we'll be able to use this macro only in this file and the macro will be attached to this file, meaning if we share this file whoever opens it from any device, will be able to use this macro. Differently from other excel workbooks when we create a file-attached macro we must save the file in ".xlsm” (excel macro enabled workbook) format.
  • New Workbook (Only): This option is like the first option but makes all the processes in a new workbook. Similarly, you must save the file in “.xlsm” format.
  • Personal Macro Workbook: This option adds the macros we'll create to a personal macro workbook and makes our macros reachable (only) for us from any excel file. When we choose this option, we don't have to save the current file in a different format since the macro isn't attached to it. Also meaning that when we share this file, the person we shared the file with won't be able to use this macro tool.

Macro Buttons

Let's take a situation where we have a dataset summarized by a pivot table. Then we created a chart to represent the data better. And we want to add button slicers to the chart, but our data is complicated that it's exceeding slicer's analysis capabilities. So, we want to assign this complicated analysis process to a button. There are two main ways to make this:

  • Developer Tab: Firstly, we record the process in a macro. Secondly, we go to Developer> Controls> Insert> and click to Button. Then we assign the macro to this button.
  • Custom Button: Similarly, we record the process in a macro. Then we go to Insert> Illustrations> Shapes and insert a shape we chose. We add text to it by right click & edit text. Then we make a right click and choose assign macro and assign the macro we recorded earlier.

Extending & Editing Macros

Deleting Unnecessary Steps:

If we have recorded a macro but it contains some unnecessary steps, we can delete these steps using VB. Firstly, we go to Developer> Visual Basic or Alt+F11. Then we open the module from current workbook's (file's name will be like "????.xlsm / .xlsx") or personal macro file. Here we'll see the code of the macros that are attached to this workbook. So, we can delete unnecessary lines from here.

  • Note: If you are not familiar with VB code be careful doing changes on the codes, otherwise you can harm them.
  • When we want to extend the processes that a macro is doing, we must go to VB code and add the codes of the additional process to the module. But this might be hard for those who aren't familiar with VB codes.
  • Here is an easier way to do it:
    We record (Be sure that you recorded it in the same place with the first macro.) the additional process as a new macro. Then we open these macros' module. We will see the codes of both macros. Cut the code of the second macro and paste it in an appropriate place between first macro's codes, then delete additional lines of the second macro.

Advanced Pivot Tables

Custom Pivot Table Analyses:

  • If we want to use an analyze method that doesn't exist in pivot table operations, we can add our custom formula from Analyze>Fields, Items & Sets> Calculated Field.
  • For example, we are analyzing dataset of a project made on the people of a country. We have population and target numbers of each subdistrict, and we have a column that calculates the coverage by dividing the target to the population. When we create the pivot table and put the governates as rows and coverage as value we will either get the sum of the coverage or the average* (not actual) coverage in this governate. So, we won't be able to reach the actual coverage for the governates.
  • Here we create a custom analyze method. We choose the columns that will be divided <=Target / Population> and press Enter.
  • Now we have the actual coverage number for governates.
  • *: we have 2 groups. First: 100 people and we covered 99 of them the coverage is 99%. Second: 1000 people and we covered 500 of them the coverage is 50%. If we take the average the result will be 74% while the actual coverage is 54%.
  • Another example: We have sales data for some products. But sales numbers are separated into 3 fields that represent the size. We want to get the total number of sales for each product. We can do this by adding a new column to the source dataset. And alternatively, we can do it using custom formulation.
  • If we want to edit or delete custom formulation, we can do this by going back to Fields, Items & Sets.

Advanced Filter

  • This feature makes the filtration process faster and easier. We can reach it from Data> Sort & Filter> Advanced.
  • Firstly, we copy the headers of our table and paste them in a different place. Then under new headers we type filtration criteria. (We can use ">,<" signs.)
  • Then we start an advanced filtration process. In the first cell we place the range while in the second cell we place the criteria part.
  • Tip: We can duplicate a header cell if we want to add multiple criteria for this header.
  • We can add more than one row in the criteria range. Adding a criterion with the same row of other criteria means “and” while adding it to another row means "or".

Automating Advanced Filter:

  • Create a new sheet and paste your headers there.
  • Start and advanced filtration process and select "Copy to another location". Select the ranges and finish the process. (Be sure that the active sheet is the new one.)
  • Then record a macro and repeat the advanced filter process and select the exact same ranges.
  • Go to Developer tab and add a button (from insert). Assign the macro to it.
  • Now whenever you update the filtration range and press this button the filtration will be applied.

Reference Cells, Columns & Rows

When we are formulating a large range and using auto fill to do so, we might want to fix or lock a cell / column / row in the formulation. We use references in this situation. There are three reference types in excel:

  • A relative reference is a cell address without any $ sign, such as A1. It means that the reference will adjust both the row and column according to where the formula is located.
  • An absolute reference is a cell address with a $ sign before both the row and column coordinates, such as $A$1. It means that the reference will always point to the same cell, no matter where the formula is located.
  • A mixed reference is a cell address with a $ sign before either the row or the column coordinate, but not both, such as $A1 or A$1. It means that only one part of the reference will stay fixed, while the other part will change relative to the position of the formula.
  • Using mixed reference, we can make a multiplication table: <=$A2*B$1> (Then drag the cell down and right.)

INDEX & MATCH Functions

Index (Array, Row Number, Column Number):

  • Index function is a developed form of lookup functions. Firstly, we define the range then row and column numbers of the value we are looking for.
  • Note: If you include in index function's array the headers you should include corner cell in match function's array. Otherwise, the numbers won't match.
  • Defining row and column numbers might take too long so here match function takes place.

Match (Lookup Value, Array, Match Type):

  • This function prompts the column or the row number of the lookup value in the defined array. Match Type is 0: exact same / 1: smaller values / -1: bigger values.
  • So, we can use this function to define row and column numbers in the index function.
    <=Index (C5:H10, Match (“T-shirt”, B5:B10,0), Match (2020, C4:H))
  • But it is still hard to determine a product's name and sale year in a given data set. So, we can select two cells for match functions and whatever we type in these cells will lookup value for match functions.
  • Tip: If you define row numbers in index function you don't have to define columns. If you don't, the default value will be 1.

OFFSET Function

Offset (reference, rows, columns, [height], [width])

  • Reference: The base cell for the formula.
  • Rows: Destination's coordinates in y axis according to the reference. (+: down / -: up)
  • Columns: Destination's coordinates in x axis according to the reference. (+: right / -: left)
  • Height: height of the range it will bring. (Default: 1) - optional
  • Width: width of the range it will bring. (Default: 1) - optional

We use this formula to bring a specific cell / range. Mostly, it's used with other formulas. Examples:

  • =OFFSET(C3,MATCH(J7,B4:B15,0),0)
    This formula starts from C3. Then it looks for the y axis coordinates of the specified value in J7 using match function. And doesn't change column (0). So it looks for the value of J7 between B4:B17 and brings the value at the right side. An alternative for vlookup.
  • =OFFSET(C3,COUNTA(B4:B20),0,-3)
    This formula starts from C3. Then counts the cells with values (goes to the last filled cell) and brings the last 3 cells.
  • Note: In excel formulas if a variable is in square brackets, it means that it is optional.

Dynamic Drop-Down Lists with Offset

We have three countries, and each country has some governorates, and each governorate has some cities in it. We want to make a drop-down list that changes according to the country & governorate values. Here are the steps:

  • Firstly, create the main table that will be composed of 3 columns: country / governorate / city.
  • Secondly, create a table of these values in a new sheet. Each country's name should be repeated as much as it's governorates name and each governorate should be repeated as much as it's cities name.
  • Copy countries' column to another place and remove the duplicates. Sort the new table from A to Z. Then go to the main table and create a drop-down list (Data> Data Validation> List) in the country column sourcing the new country column in sheet 2.
  • Copy country and governorate columns from source table to another place then remove the duplicates. Sort the new table from A to Z. Then, update cell values in this formula according to your file and use it to make dynamic list for governorates: 
    =OFFSET(Sheet2!$D$1,MATCH(Sheet1!$C4,Sheet2!$C$2:$C$9,0),0,COUNTIF(Sheet2!$C$2:$C$9, Sheet1!$C4),1)
  • Copy governorate and city columns to another place and remove the duplicates (to be sure) and sort the table from A to Z. Then, update cell values in this formula according to your file and use it to make dynamic list for cities: 
    =OFFSET(Sheet2!$G$1,MATCH(Sheet1!D4,Sheet2!$F$2:$F$17,0),0,COUNTIF(Sheet2!$F$2:$F$17,Sheet1!D4))
